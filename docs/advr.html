Phase 1: Architect - The Verbose Action

Generate Your Core Documents

Use a foundational LLM like Claude, Gemini, or ChatGPT for this initial, broad-stroke generation. The goal is speed and comprehensive ideation, not perfect prompt engineering.

1. The "Good Enough" Prompt:

Your prompt should be descriptive but not overly structured. Its purpose is to generate three key documents: a Product Requirements Document (PRD), an Architecture Guidance doc, and a Technical Requirements Document (TRD).

2. Example Prompt:

"Act as a senior software architect. I need to build a user satisfaction solution, where the implementation should adopt an API first strategy. We are an organization trying to enhance our core EMR's user experience and want to capture a simple way for providers to give feedback, with little to no friction... this is about allowing providers to communicate their sentiment and perceived value of our software. It has to be HIPAA compliant, secure scalable, distributed platform using an existing SQL database. The service objectives are adapted from... happy-or-not.com The API should be implemented using Node.js, Express, and PostgreSQL. The UI stack should be React/NextJS/Tailwind Css... We intend to deploy the solution to an AWS environment... I need you to generate a PRD, a first draft of an Architecture Guidance and Standards and a TRD based on the two others as markdown files. Based on these documents I want you to generate simple, separate .md files with prompts to generate the UI, the API and SQL artifacts that I can use as next steps to actually generate the code for each of those layers"

3. Your Task After Generation:

The AI will generate the requested documents. Your job is to review, correct, and edit them. The output won't be perfect, but it will be a comprehensive starting point. Refine it until you have a "good enough," complete version of your PRD to guide the next phase.

Phase 2: Direct - The Verbose Action

Execute with a Coding Agent

Now, take your refined documents from the Architect phase and use them to direct a specialized AI Agent Coding Assistant (like an IDE extension or a dedicated tool). The key is to provide focused, high-quality context.

1. The Refined, Context-Rich Prompt:

Instead of a broad request, you will now provide the specific documents you curated. This is a much more precise instruction for a tool designed to write code.

2. Example Prompt:

"Generate the API implementation ONLY for the application described in the PRD attached. Follow and adhere to the architecture guidance and standards provided, and implement following the indications in the TRD attached. Do NOT generate any UI components. Generate all unit tests, IaC and instructions to build an initial simple CI/CD pipeline using Github Actions."

3. Your Task During Generation:

The coding agent will now generate the application code based on your precise instructions and documentation. This is where the blueprint you created in Phase 1 pays off, leading to higher quality, more consistent output.

Phase 3: Validate - Practical Steps

You Are the Quality Gate

This phase is self-explanatory but critical. Never assume the AI's output is perfect. Your expertise is required to ensure quality and correctness.

  • Code Review: Read through the generated code. Does it make sense? Does it follow the patterns from your architecture document? Are there obvious bugs or security flaws?
  • Run the Code: Execute the generated files. Do they compile/run without errors?
  • Test the Logic: Run the unit tests generated by the AI. Do they pass? Do they actually test the business logic correctly? Write your own tests if necessary.
  • Integrate and Verify: Add the new code to your main project branch. Does it break anything? Does the application as a whole still function as expected?

Phase 4: Refine - Rinse and Repeat

The Engine of Progress

This is the "Rinse and Repeat" step. Your actions here are determined by the outcome of the Validate phase.

If the code is buggy or incorrect:

Return to the Direct phase. Craft a new, more specific prompt to address the issue. Don't try to fix large chunks of bad code manually; use the AI to regenerate the problematic section with better instructions.

Example Fix Prompt: "The database connection logic you provided doesn't handle retries. Please rewrite the connection function to retry the connection 3 times with a 5-second delay between attempts."

If the code works as expected:

Congratulations! Merge it. Now, return to the Architect phase. Look at your PRD, pick the next feature or user story, and begin the A-D-V-R cycle all over again.
