The A-D-V-R Cycle

Your High-Speed Workflow for the AI Hackathon

You are the Architect. The AI is your Apprentice.

1. ARCHITECT

Create the Blueprint

Your Goal:

Create a clear, simple plan for a single feature. Think in 1-2 hour chunks of work, not the whole project.

Your Action:

Define a micro-goal, sketch the components (functions, files, tech), and use the AI as a brainstorming partner to outline the structure.

Prompt Example:

"Act as a senior software architect. I need to build a user registration endpoint using Node.js, Express, and PostgreSQL. Suggest a simple file structure and the main functions I'll need."

2. DIRECT

Craft the Prompt

Your Goal:

Delegate one specific, well-defined coding task to your AI assistant. Offload the tedious, systematic work.

Your Action:

Pick the smallest piece from your blueprint. Craft a precise prompt with just enough context for the AI to succeed.

Prompt Example:

"Using Node.js and Express, write the code for `server.js`. It should import express, create an app, define port 3000, and include a basic health check route at `/`. Add comments explaining each part."

3. VALIDATE

Perform the Code Review

Your Goal:

Critically review, test, and integrate the AI's output. You are the quality gate. Never trust AI code blindly.

Your Action:

Read every line to ensure it aligns with your blueprint. Test the code in isolation first, then integrate it into your project and verify it works with other components.

Your Motto:

"Trust, but verify."

4. REFINE

Iterate and Improve

Your Goal:

Based on your validation, either fix issues or move on to the next task. This is the loop that drives all progress.

Your Action:

If it's broken: Go back to the DIRECT phase with a new prompt to fix the specific issue.
If it works: Go back to the ARCHITECT phase and start a new cycle on the next feature.

Prompt Example (Fix):

"The previous code failed because the port was hardcoded. Modify it to use an environment variable `PORT` with a fallback to 3000."

Architect → Direct → Validate → Refine → Repeat

Use this cycle to build momentum, maintain quality, and win the hackathon.
