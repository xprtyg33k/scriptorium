What Is Upwind?

Upwind is a cloud-native application protection platform (CNAPP) that differentiates itself by providing runtime intelligence — real-time, inside-out visibility into how applications actually behave in production — rather than relying on periodic snapshots or static configuration scans alone.

Core thesis: Security decisions are only as good as the context behind them. Upwind instruments workloads via an eBPF sensor on the kernel to deliver continuous, low-overhead telemetry — enabling prioritization based on what is actually exploitable at runtime, not merely what could be.

Origin & Philosophy

Founded by the team behind Spot.io (acquired by NetApp for its cloud cost-optimization technology), Upwind was born from a real pain point: after acquisition, NetApp flagged hundreds of vulnerabilities — but the existing CNAPP tools couldn't distinguish genuine, exploitable risk from noise. The founders built Upwind to solve exactly that problem — actionable, runtime-contextualized security.

Key Capabilities

1. Runtime Exploitability Analysis

Goes beyond "does this CVE exist" to answer: is the vulnerable package loaded, is the specific function invoked, is the workload internet-exposed, and is there a known exploit path? This dramatically reduces alert fatigue and focuses remediation on real risk.

2. Application-Aware Topology & API Discovery

Automatically maps service-to-service communication, discovers API endpoints, and detects sensitive data in transit — providing a live dependency graph grounded in actual traffic, not static config.

3. Real-Time Threat Detection

Correlates behavioral events (e.g., SQL injection attempts, log4shell exploitation, reverse shells, credential access) into unified attack narratives — minute-by-minute, not after-the-fact.

4. Agentic Remediation (Red / Blue / Green Agents)

  • Red Agent — Automated attack-surface validation; pen-tests exposure paths
  • Blue Agent — Defensive posture assessment; correlates events into threat narratives
  • Green Agent — Generates lowest-effort remediation paths for prioritized issues

5. Code-to-Cloud (Shift Left)

  • Image scanning with runtime-informed risk context
  • PR-level recommendations: Deploy, Deploy with Caution, Do Not Deploy
  • CVE differential analysis per build (new vs. resolved vulnerabilities)
  • Optional build-fail gates or audit-only mode

6. Operational Integration

Designed to push value into existing workflows — Slack, Teams, Jira, ServiceNow, SIEM — so teams receive actionable notifications without living inside another dashboard.

The Problem It Solves

Traditional CNAPP solutions take periodic snapshots of cloud environments (every 6–24 hours) and generate lists of potential vulnerabilities with no context as to whether they're actually exploitable. This creates two compounding problems:

  • Alert fatigue: Teams drown in CVE notifications that may have zero real-world impact
  • Blind spots: Active threats occurring between scans go undetected until post-incident analysis

Upwind eliminates both by continuously observing runtime behavior and only escalating issues that are demonstrably exploitable — the package is loaded, the function is invoked, and there is a viable attack path.

How It Works — Technical Foundation

  • eBPF sensor — Hooks into the Linux kernel without kernel extensions; avoids the overhead and crash risk of traditional agents (no kernel panic scenarios like CrowdStrike's 2024 incident)
  • Application-aware topology — Maps live service-to-service communication, API endpoints, and data flows automatically from observed traffic
  • Runtime exploitability engine — Correlates CVEs with actual package usage, function invocation, network exposure, and exploit availability
  • AI agents (Red/Blue/Green) — Automate pen testing, defensive correlation, and remediation generation
  • Code-to-Cloud pipeline — Scans builds with runtime context, provides deploy/caution/block recommendations at the PR level

What This Solves for InfoSec

  • Closes the scan-to-runtime gap: Today we know a CVE exists in an image. Upwind tells us whether it's loaded, the vulnerable function is being called, and there's an internet-facing path to it. This is the difference between "possible risk" and "confirmed exposure."
  • Real-time incident detection: The Blue Agent correlates events (SQL injection, reverse shell, credential harvesting, log4shell) into unified attack narratives as they happen — not hours later from log analysis.
  • Zero-day discovery: Upwind is actively publishing zero-days found through runtime behavioral observation — catching what no CVE database has cataloged yet.

How It Complements (Not Replaces) Current Tools

  • CrowdStrike — Remains your EDR for endpoints. Upwind operates at the cloud-native workload layer (containers, EKS, serverless) where CrowdStrike has limited depth.
  • Rapid7 — Continues as periodic pen-test validation. Upwind's Red Agent provides continuous, automated attack-surface probing between assessments — raising your confidence level going into each pen test.
  • Pen testing scope: With continuous exploitability validation, you can better target pen-test efforts on areas of genuine uncertainty rather than broad surface sweeps.

Runtime Topology & API Discovery

Upwind automatically maps how services communicate in production — not based on config files or Terraform, but on observed traffic:

  • Service-to-service call graphs with actual traffic patterns
  • Automatic API endpoint discovery with sensitive data detection in payloads
  • Drift detection — does runtime behavior match architectural intent?

eBPF — Why This Is Production-Safe

Technical note: eBPF (Extended Berkeley Packet Filter) provides kernel-level observability via verified, sandboxed programs — no kernel modules, no extensions to update, no user-space agent competing for CPU/memory. This avoids the class of problems that caused CrowdStrike's 2024 global outage. Resource overhead is minimal and predictable.

  • No kernel extensions — no panic risk
  • No heavy user-space agent — safe for latency-sensitive production workloads
  • AWS partnership — designed and validated for EKS/ECS/Lambda environments
